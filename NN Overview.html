⚙️ 1. DNNRegressor

WHAT A multi‑layer fully‑connected neural network for regression (TensorFlow Estimator).

WHY Captures non‑linear relationships between features (latitude, income, rooms…) and house value – linear models underfit such spatial & socio‑economic data.

HOW Forward pass: \( y = W_n \cdot \sigma(...W_2 \cdot \sigma(W_1 \cdot x + b_1)...) + b_n \).
Hidden units: e.g. [10,2] or [10,10] with ReLU activation (implicit).
Output: single unit with linear activation (regression).

dnn_regressor = tf.estimator.DNNRegressor( hidden_units=[10, 2], optimizer=GradientDescentOptimizer(..) )

📉 2. Gradient descent + clipping

WHAT Optimizer: GradientDescentOptimizer(learning_rate) with gradient norm clipping (5.0).

WHY Standard optimizer for understanding fundamentals. Clipping prevents exploding gradients — common in deep nets with outliers.

HOW \( \theta_{t+1} = \theta_t - \eta \cdot \text{clip}(\nabla_\theta \mathcal{L}, -5, 5) \)
Loss function: mean squared error (regression).

RMSE = √( 1/n Σ(yᵢ – ŷᵢ)² )

🧩 Deep dive — algorithms & rationale

🔷 What is the core algorithm?

DNNRegressor from TensorFlow Estimators. It implements a feedforward network with configurable hidden layers. For each layer: \( h_{i+1} = \text{ReLU}(W_i \cdot h_i + b_i) \). The final layer omits activation, yielding a continuous prediction. Training uses mini‑batch SGD with backpropagation.

Why neural nets? Housing prices are driven by complex interactions (e.g. location (lat/lon), income, rooms). A linear model would miss these; deep nets approximate any continuous function given enough capacity.

🔶 Why rooms_per_person?

feature synthesis The notebook explicitly creates rooms_per_person. While neural networks can learn ratios, explicit feature engineering reduces burden on early layers and often improves convergence. It encodes density / crowding, a known driver of house value (more rooms per person → more spacious → expensive).

🔷 Why gradient clipping by norm (5.0)?

Exploding gradients are common in deep networks, especially with MSE loss and outliers in features like total_rooms or population. Clipping ensures stable updates; clip_gradients_by_norm scales back gradients if their L2 norm exceeds 5.0.

🔶 How is the model evaluated?

Using sklearn.metrics.mean_squared_error then sqrt. The notebook computes RMSE on training, validation, and test. RMSE is in same unit as target (thousands of dollars). Example: RMSE = 80 means typical error ~$80,000. Test evaluation after tuning reveals final performance.

test_predictions = dnn_regressor.predict(..) rmse = sqrt(mean_squared_error(test_predictions, test_targets))

✅ Summary — what we learned

  • 📌 Data preprocessing – synthetic features + train/val/test split are crucial for unbiased eval.
  • 📌 DNN architecture – hidden units and depth affect under/overfitting; validation curve guides choice.
  • 📌 Optimizer – vanilla SGD with gradient clipping works; but Adam might converge faster (not shown).
  • 📌 Input pipeline – TensorFlow Dataset API feeds data efficiently; batching/shuffling vital.
  • 📌 Overfitting control – no explicit regularisation (dropout/L2) in this notebook, but early stopping via period monitoring implicit.
  • 📌 Reproducibility – random permutation fixed but seeds not set, runs may vary slightly.
